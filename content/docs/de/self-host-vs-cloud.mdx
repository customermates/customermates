---
title: Self-Host vs Cloud
description: Welches Customermates-Deployment zu dir passt — unsere EU-gehostete Cloud oder deine Infra.
---

**TL;DR** — Cloud ist schneller loszulegen und wir kümmern uns um alles. Self-Host gibt dir Datenhoheit und Kostenkontrolle, du betreibst aber selbst. Beide haben denselben Funktionsumfang.

## Die kurze Antwort

| Wenn du… | Nimm |
|---|---|
| Heute noch mit dem CRM arbeiten willst | **Cloud** |
| Ein Team von < 20 ohne harte Data-Residency-Anforderung | **Cloud** |
| Das CRM auf Private-Infra für Compliance brauchst | **Self-Host** |
| Long-term weg von Pro-Seat-Pricing willst | **Self-Host** |
| Agentur mit mehreren Mandanten bist | **Self-Host**, eine Instanz pro Mandant |
| Zum Projekt beitragen willst | **Self-Host** lokal + Cloud für echte Arbeit |

## Was du bekommst

| | Cloud | Self-Host |
|---|---|---|
| Setup-Zeit | 2 Minuten | ~15 Minuten |
| Infra, die du managst | keine | Docker, Postgres, Proxy, TLS, Backups |
| Updates | automatisch | `docker compose pull && up -d` |
| EU-gehostet | ✓ | wo du willst |
| Backups | automatisch täglich | du konfigurierst |
| Support-SLA | inkl. in Paid-Plans | Community |
| Enterprise-Features (SSO, Audit-Log) | Paid-Plan | Paid-License-Key |
| Pricing | pro Sitz | kostenlos, nur Enterprise-Lizenz zahlen |

## Datenhoheit

Die Cloud speichert Daten in unserer EU-Region, DSGVO-konform, mit üblicher Verschlüsselung at-rest und in-transit. Wenn deine Compliance verlangt, dass Daten auf deiner Infra liegen: Self-Host.

## Kostenform

- **Cloud** ist vorhersagbar pro Sitz. Linear skalierend.
- **Self-Host** ist kostenlos für den Core + was du für Infra zahlst (ein kleiner VPS trägt Hunderte User).

Für ein 5er-Team ist Cloud meist günstiger, wenn du die Zeit fürs eigene Postgres-Backup einrechnest. Für ein 50er-Team lohnt sich Self-Host schnell.

## Wechseln zwischen beiden

Export von einem, Import ins andere. Identisches Datenmodell. Kein Lock-in in beide Richtungen.
