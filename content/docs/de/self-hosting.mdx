## Customermates self-hosted mit Docker Compose starten

Diese Seite zeigt, wie du eine self-hosted Customermates-Installation mit Docker Compose und PostgreSQL aufsetzt.
Wenn du noch zwischen Deployment-Modellen vergleichst, gehe zuerst zu [Self-Hosted vs Cloud](/docs/self-host-vs-cloud).

Self-hosted Deployments laufen standardmäßig in der Community Edition. Die kostenpflichtigen Enterprise-Features (Audit Logging, Single Sign-On, AI Agent, Whitelabeling) sind in self-hosted Deployments nicht aktiv.

## Voraussetzungen

Vor dem Deployment solltest du Folgendes bereit haben:

- ein VPS mit installiertem Docker Engine und Docker Compose
- eine Domain, die du auf deinen Server zeigen lassen kannst
- Zugriff zum Anlegen von DNS-Eintraegen

Fuer produktive Nutzung solltest du einen Reverse Proxy mit HTTPS-Terminierung vor den App-Container setzen.

## Installationsablauf

Lade die benoetigten Dateien herunter und fuehre das Setup-Skript aus:

```bash
mkdir customermates && cd customermates
curl -O https://raw.githubusercontent.com/customermates/customermates/main/docker-compose.yml
curl -O https://raw.githubusercontent.com/customermates/customermates/main/.env.selfhost.template
curl -O https://raw.githubusercontent.com/customermates/customermates/main/scripts/selfhost-setup.sh
mv .env.selfhost.template .env
# .env mit deinen Werten bearbeiten
chmod +x selfhost-setup.sh
./selfhost-setup.sh
```

Das Setup-Skript:

1. erforderliche Inputs und Umgebungswerte validieren
2. das vorgefertigte App-Image aus der Container Registry pullen
3. PostgreSQL starten und Datenbank-Migrationen ausfuehren
4. App- und Webhook-Worker-Container starten

Wenn du Konfigurationswerte anpassen musst, aktualisiere deine `.env` und fuehre das Setup-Skript erneut aus.

## Initiale Startup-Checks

Nach dem Setup pruefst du, ob die Container laufen:

```bash
docker compose ps
```

Danach kannst du die App-Logs verfolgen, bis die Anwendung als bereit gemeldet wird:

```bash
docker compose logs -f app
```

Fuer Datenbank-Diagnosen:

```bash
docker compose logs -f postgres
```

## Erster Login und Signup

Sobald die App im Browser erreichbar ist:

1. Oeffne deine konfigurierte Domain.
2. Erstelle dein erstes Konto oder melde dich an.
3. Schließe Onboarding und initiales Company-Setup ab.

Nach dem ersten Login geht es weiter mit [Installation verwalten](/docs/managing-your-installation) fuer Updates, Restarts und Troubleshooting.
Fuer die operativen Unterschiede zwischen den Deployment-Modellen siehe auch [Self-Hosted vs Cloud](/docs/self-host-vs-cloud).
