**TL;DR** — MCP für KI-getriebene Reads/Writes. OpenAPI für gebaute Integrationen. Webhooks für Push-Notifications bei Änderungen. n8n für visuelle Workflows. Die meisten Teams nutzen mehrere gleichzeitig.

## Entscheidungs-Matrix

| Ich will… | Nimm |
|---|---|
| Claude/ChatGPT aktualisiert mein CRM im Chat | [MCP](/docs/mcp) |
| Custom-Integration in Code bauen | [OpenAPI](/docs/openapi) |
| Auf CRM-Änderungen in meinem Stack reagieren (Slack, Sequencer, BI) | [Webhooks](/docs/webhooks) |
| Workflows visuell ohne Code komponieren | [n8n](/docs/n8n) |
| Zwei-Wege-Sync mit Zapier-lastigem Stack | [Webhooks](/docs/webhooks) + REST |

## MCP

Model-Context-Protocol-Endpoint unter `/api/v1/mcp`. Dein KI-Client entdeckt 57 CRM-Tools, ruft sie per Name auf, bekommt strukturierte Fehler zurück. Auf Zuverlässigkeit auch mit schwächeren Modellen ausgelegt: schmale Schemas, inline Enum-Hinweise, Relationship-Safety.

[**MCP-Übersicht →**](/docs/mcp) · [**Tool-Katalog →**](/docs/mcp-tool-catalog)

## REST (OpenAPI 3.1)

Kanonische HTTP-Oberfläche. Jede CRM-Operation ist ein dokumentierter Endpoint mit Request- und Response-Schema. Nutze diesen, wenn du Integrationscode schreibst und weißt, was du brauchst.

[**OpenAPI-Referenz →**](/docs/openapi)

## Webhooks

Jeder Entity-Write emittiert ein Domain-Event. Abonniere per URL, wir POST-en JSON dorthin bei jeder Änderung. Signiert per HMAC-SHA256, wenn du ein Secret setzt. Deliveries geloggt und re-sendable.

[**Webhooks →**](/docs/webhooks) · [**Event-Katalog →**](/docs/webhook-events)

## n8n

Für Teams, die visuelle Automation bevorzugen. Die Customermates-n8n-Nodes wrappen die MCP- und REST-Oberflächen mit Drag-and-Drop-Triggern und -Actions.

[**n8n →**](/docs/n8n)

## Welche Kombi passt

- **Solo-Operator mit KI-first-Workflow:** MCP für Writes, Webhooks in Slack oder E-Mail für Benachrichtigungen. Kein REST nötig.
- **Kleines Sales-Team:** Webhooks in den vorhandenen Sequencer, MCP für die KI des Founders, gelegentlich OpenAPI-Skripte fürs Aufräumen.
- **Integrator, der auf Customermates baut:** OpenAPI für die Masse, Webhooks fürs Event-Driven, MCP wenn dein Produkt selbst Agent-getrieben ist.

## Authentifizierung

Alle vier nutzen denselben API-Key. Siehe [API-Keys](/docs/api-keys).
