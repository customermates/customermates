Wenn du den Stack noch nicht installiert hast, beginne mit [Self-Hosting Erste Schritte](/docs/self-hosting).

Self-hosted Betrieb nutzt die Community Edition. Enterprise-Features erfordern eine separate Lizenz.

## Aktualisieren

```bash
./selfhost-update.sh
```

Pullt das neueste App-Image, fuehrt `prisma migrate deploy` aus und startet App- und Webhook-Worker-Services neu. Pruefen mit `docker compose ps`.

## Neustarten

```bash
./selfhost-restart.sh
```

Startet den Stack ohne Migrationen neu. Verwenden nach `.env`- oder Konfigurationsaenderungen.

## Logs und Troubleshooting

```bash
docker compose logs -f app
docker compose logs -f postgres
docker compose ps
docker compose exec app sh
```

## Alle Daten zuruecksetzen

```bash
./selfhost-reset.sh
```

Erfordert explizite `RESET`-Bestaetigung vor dem Loeschen der Volumes. Nicht umkehrbar.

## Backups

Fuer produktive Deployments:

- PostgreSQL-Daten regelmaessig sichern
- Restore-Prozesse in einer Nicht-Produktionsumgebung testen
- `.env` und Secrets aus der Versionskontrolle heraushalten
