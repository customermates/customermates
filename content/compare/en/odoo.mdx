## What is Odoo CRM?

Odoo CRM is one module of Odoo, a Belgian business suite with 80+ integrated applications. The CRM module covers lead management, pipeline, quotes, and reporting, and ties seamlessly into other Odoo modules like Sales, Accounting, and Inventory. That suite character is Odoo's strength and its defining trait: choosing the CRM means choosing the platform.

Odoo splits two editions. The Community Edition is licensed under LGPLv3 and lives on [GitHub](https://github.com/odoo/odoo), free to self-host but missing many Enterprise modules. The Enterprise Edition is proprietary and ships in three plans: One App Free (one app, free forever), Standard ($24 / €19.90 per user per month yearly, $30 / €24.90 monthly), and Custom ($37 / €29.90 yearly, $47 / €37.40 monthly). REST API access for external integrations is gated to the Custom plan.

## What is Customermates?

Customermates is a CRM that updates itself if you already use [Claude](/blog/chatgpt-summarize-notes), ChatGPT, or Codex. The pipeline moves, follow-ups get drafted, and contact notes stay current because the agent types. Underneath that outcome layer sits a modern Pipedrive replacement: five entities (Contacts, Organizations, Deals, Services, Tasks), two views (Table and Kanban-style Card), eight custom field types, and a full REST API. Open source under AGPL-3.0, free to self-host via Docker, EU-hosted in the cloud from €9 per user per month on the yearly plan.

The mechanism that makes the AI integration real: a built-in MCP server with 57 tools, an n8n community node, 15 webhook events, and the OpenAPI 3.1 spec at `/api/v1/openapi`.

## Setup and onboarding

Odoo Online provisions in minutes, but configuring real sales workflows usually takes longer. Many companies bring in an Odoo implementation partner because the suite has depth that isn't trivial to navigate without prior experience: 80 modules, Studio customization, module dependencies. Self-hosting the Community Edition is possible but loses many Enterprise-only modules.

Customermates is ready in two minutes. The Card view groups by any single-select column, so any entity (deals, contacts, organizations, tasks) becomes a Kanban board. Self-hosting runs through `docker compose up -d` with three containers (App, PostgreSQL 16, Webhook Worker). No implementation partner needed.

## CRM depth

Odoo's CRM module is mature on lead management. Lead mining, AI-based lead scoring on historical data, UTM tracking, deduplication, GeoIP, and lead routing are all native. Teams generating high lead volume from marketing channels and needing systematic scoring and distribution get a workflow Odoo has been refining for years.

Customermates ships five entity types as first-class citizens: Contacts, Organizations, Deals, Services, and Tasks. They link bidirectionally through one `link_entities` operation. Custom fields are unlimited and typed (eight types: plain, date, datetime, currency, singleSelect, link, email, phone), available from day one rather than as a Studio upsell. Lead mining and enrichment aren't built in but plug in through the MCP server, n8n, or Apify in minutes because the API exposes every operation.

## AI: lead scoring vs. an agent that operates the CRM

Odoo's AI focuses on one job: lead scoring. The platform analyzes historical data and predicts which leads will convert. Useful for sales teams with high lead volume, but conceptually narrow. The AI scores. It doesn't operate.

Customermates was built for AI agents to run the CRM directly. The MCP server exposes 57 tools covering every CRM operation: create deals, update notes, link entities, configure custom columns, run filters, list webhooks. Point Claude or Codex at `/api/v1/mcp` with your API key and the model reads and writes the CRM as if it were a human user. The same layer powers a [native n8n community node](/blog/n8n-for-beginners-build-your-first-workflow) and 15 webhook events.

The difference matters when the CRM should update itself from emails, calls, transcripts, or chats rather than waiting for a human to score leads. Odoo scores leads. Customermates lets your agent operate the entire CRM.

## Open source and data sovereignty

Odoo splits cleanly between Community (LGPLv3, free) and Enterprise (proprietary, paid). The Community Edition covers core CRM but many specialty modules are Enterprise-only. Odoo Online runs on Odoo SA infrastructure in Belgium. Real data sovereignty in your country requires the Custom edition with self-hosting or the Community Edition on your own servers.

Customermates is licensed under [AGPL-3.0](https://www.gnu.org/licenses/agpl-3.0.html). The full source code lives on [GitHub](https://github.com/customermates/customermates). Self-hosting via `docker compose up -d` starts three containers. The cloud version runs in Germany, GDPR-native by default. If the project ever stops, your data and the code are still yours.

The license difference is subtle but real. AGPL-3.0 forces SaaS providers to publish source changes when they host the product. LGPLv3 (Odoo Community) allows proprietary extensions without giving back. Both are open source, but AGPL is friendlier to the community.

Customermates leans on open interfaces from the first plan. Webhooks cover every CRUD operation across all five entities, so external systems stay in sync without polling.

## Frequently asked questions

**Is Odoo CRM really free?**
Partly. Odoo offers a One App Free plan that lets you use a single app forever, free, with unlimited users (so just CRM works). The moment you need a second app, you fall back to Standard at €19.90 per user per month yearly. The Community Edition is also free but ships without many Enterprise modules. Customermates offers a 3-day Pro trial without a credit card and is free to self-host as open source under AGPL-3.0.

**Is Odoo good for CRM?**
Yes, especially for companies already running the Odoo suite. Lead management, pipeline, quote generation, and AI lead scoring are mature. Weak points: no native AI agent integration, REST API only on the Custom plan, and the CRM is designed as a module within a large suite, which means overhead for teams that don't need ERP or accounting.

**What is the best alternative to Odoo CRM?**
Depends on the use case. Companies running ERP suite functionality stay with Odoo. Teams looking for a focused CRM with native AI agent integration find Customermates as an open-source alternative starting at €9 per user per month with full API from plan one. For other CRM alternatives, see the [Pipedrive alternative](/compare/pipedrive-alternative) and [HubSpot alternative](/compare/hubspot-alternative) pages.

**Is Odoo a competitor of Salesforce?**
Indirectly. Odoo positions itself as an integrated suite against the Salesforce ecosystem of Sales Cloud, Service Cloud, Marketing Cloud, Pardot, and many add-ons. They cover similar functional ground but with different pricing models. Salesforce is generally pricier and more implementation-heavy. Odoo is cheaper per user but carries its own suite complexity.

**Can I self-host Odoo CRM?**
Yes, through the LGPLv3 Community Edition or through the Enterprise Custom plan with self-hosting enabled. The Community Edition runs on your infrastructure but only covers a subset of Enterprise modules. Customermates is self-hostable in full functionality under AGPL-3.0 via `docker compose up -d`.

**Does Odoo CRM have an API?**
Yes, but only on the Custom plan (€29.90 per user per month yearly). The Standard plan does not include external REST API access. Customermates ships a full REST API with OpenAPI 3.1, an MCP server with 57 tools, and 15 webhook events starting at the €9 Pro plan.
